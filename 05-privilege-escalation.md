# Privilege Escalation & Scheduled Tasks (Level 21–26)

This section focuses on how tasks run automatically in the system and how misconfigurations can be used to gain access.  
It introduces cron jobs, script execution, and how permissions can be leveraged to move between users.

---

## Level 21

**Goal:**  
Find how a password is being generated or stored using scheduled tasks.

**Concept:**  
Cron jobs.

**Approach:**  
Explored cron directories (like `/etc/cron.d/`) to identify scheduled scripts and understand what runs automatically.

**Why it works:**  
Cron jobs execute tasks at fixed intervals. If they handle sensitive data, they can reveal useful information.

**Key Takeaway:**  
Automated tasks can expose secrets if not properly secured.

---

## Level 22

**Goal:**  
Retrieve a password generated by a cron job script.

**Concept:**  
Understanding script behavior.

**Approach:**  
Read the cron job script to see where it stores output, then accessed that location.

**Why it works:**  
Scripts often write results to files. Knowing where to look is key.

**Key Takeaway:**  
Always read scripts—they often tell you exactly where the data is.

---

## Level 23

**Goal:**  
Understand how a script generates filenames dynamically.

**Concept:**  
Predictable file naming.

**Approach:**  
Analyzed the script logic to determine how filenames are generated, then reproduced the same logic to locate the correct file.

**Why it works:**  
If a system uses predictable patterns, you can replicate them to find hidden data.

**Key Takeaway:**  
Predictability in systems can become a vulnerability.

---

## Level 24

**Goal:**  
Brute-force a service with a limited keyspace.

**Concept:**  
Brute-force attacks.

**Approach:**  
Wrote a loop/script to try multiple possible inputs until the correct one was found.

**Why it works:**  
If the input space is small and there’s no rate limiting, brute force becomes practical.

**Key Takeaway:**  
Weak protections + small input space = easy exploitation.

---

## Level 25

**Goal:**  
Access a restricted shell environment.

**Concept:**  
Escaping restricted shells.

**Approach:**  
Used available tools (like text viewers/editors) to break out of the restricted environment and execute commands.

**Why it works:**  
Some programs allow shell escapes or unintended command execution.

**Key Takeaway:**  
Restrictions are often incomplete—look for ways around them.

---

## Level 26

**Goal:**  
Leverage permissions and environment to gain access.

**Concept:**  
Privilege escalation through environment control.

**Approach:**  
Manipulated how programs were executed (e.g., controlling paths or behavior) to gain access to restricted data.

**Why it works:**  
Programs may trust environment settings, which can be influenced by the user.

**Key Takeaway:**  
Execution context matters—control over environment variables or paths can lead to escalation.

---

## Overall Takeaways (Level 21–26)

- Scheduled tasks can leak sensitive data  
- Reading scripts gives direct insight into system behavior  
- Predictable logic can be exploited  
- Weak protections allow brute-force attacks  
- Restrictions and permissions are often misconfigured
